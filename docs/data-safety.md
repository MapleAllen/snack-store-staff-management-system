# 数据安全与备份

## 存储方式

### 桌面版（Windows）

工作区数据保存在 Electron 应用数据目录下的 `workspace.json` 文件中，通过渲染层存储适配器与主进程 IPC 协调读写。文件采用原子写入策略，避免写入中断导致损坏。

自动恢复点保留在应用数据目录的 `backups/` 子目录中，由共享常量统一管理备份原因标签。每日首次启动只保留当日第一个恢复点；超过保留上限后自动清除最早恢复点。

### Web 开发预览

Web 模式仅用于本地开发，工作区数据仍保存在浏览器 `localStorage` 中。此模式不提供应用锁、桌面文件存储或自动恢复点能力，也不作为正式产品模式。

### 加密状态

- 桌面 canonical workspace file 在 v2.x 中**未加密**，依赖 Windows 账户隔离和磁盘加密（如 BitLocker）。
- 自动恢复点以明文 JSON 保存。
- 手动备份支持两种模式：**明文**（与旧版兼容）和**口令加密**（使用 PBKDF2 派生密钥 + AES-256-GCM 加密，导出后需口令方可恢复）。

## 应用访问保护

桌面版支持设置 4-6 位数字 PIN。PIN 通过 PBKDF2 哈希后保存在本机，启动或手动锁定后需输入正确 PIN 方可进入工作区。连续 5 次错误输入将触发 30 秒冷静期。PIN 设置、修改和清除均需验证当前 PIN。

当前版本不提供"忘记 PIN"后的自动恢复机制；用户需要在设置 PIN 前导出未加密备份，或通过已知 PIN 修改/清除。

## 自动恢复点

触发时机包括每日首次启动、恢复数据前、工资月结后和演示重置前。恢复流程会校验备份类型、存储键和基本工作区结构，并要求用户再次确认。

损坏文件不会出现在恢复点列表中，通过编号读取时会显式失败。

## 操作建议

- 首次录入真实数据前，确认 Windows 用户账户、磁盘访问权限和 BitLocker 状态。
- 设置 PIN 前先导出一份未加密备份保存在受控介质中。
- 每次升级、迁移设备和批量修改前导出 JSON 备份。
- 如需加密备份，使用"导出加密备份"功能并妥善保管口令。
- 将手动备份复制到受控的其他介质，不要只保存在同一块磁盘。
- 不要通过公开 Issue、聊天记录或源码仓库传输备份或口令。
- 月结后抽查正式导出与冻结快照；发现错误时记录原因后再解锁。

## 已知边界

本项目没有云端冗余、多人权限、审计服务器、远程擦除或自动更新。设备损坏、恶意软件、共享 Windows 账户和未加密备份均可能导致数据丢失或泄露。PIN 锁不能替代操作系统级账户隔离。加密备份的保护强度取决于口令强度。
