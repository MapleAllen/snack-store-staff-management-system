# 数据安全与备份

## 存储方式

### 桌面版（Windows）

桌面版工作区数据保存在 Electron 应用数据目录下的 `workspace.json` 文件中。渲染进程通过 `src/storageAdapter.js` 调用 preload API，Electron 主进程负责读取和写入文件。

`workspace.json` 使用备份信封格式，并通过临时文件写入后 `rename` 的方式降低写入中断风险。当前版本提供结构校验和损坏状态识别，但尚未提供 checksum、写前日志、多实例锁或工作区文件加密。

自动恢复点保存在应用数据目录的 `backups/` 子目录中。每日首次启动只保留当日第一个启动恢复点；恢复点数量超过保留上限后会删除最早的有效恢复点。

### Web 开发预览

Web 模式仅用于本地开发和界面调试，工作区数据保存在浏览器 `localStorage` 中。此模式不提供桌面文件存储、自动恢复点或 PIN 锁，不作为正式产品模式。

### 加密状态

- 桌面 canonical workspace file 在当前 `2.x` 中未加密。
- 自动恢复点以明文 JSON 保存。
- 手动备份支持明文 JSON 和口令加密 JSON 两种模式。
- 加密手动备份使用 PBKDF2 派生密钥和 AES-GCM；恢复时必须提供正确口令。
- PIN 锁不加密工作区文件，也不加密自动恢复点。

## 应用访问保护

桌面版支持设置 4-6 位数字 PIN。PIN 通过 PBKDF2 哈希和随机盐保存在 `lock.json` 中。启动时如果检测到 PIN，应用会先显示锁屏。连续 5 次错误输入会触发 30 秒冷静期。

当前版本没有忘记 PIN 的自动恢复机制。用户应在设置 PIN 前导出一份受控备份，并确保 Windows 账户和磁盘加密策略可靠。

## 自动恢复点

自动恢复点触发时机包括：

- 每日首次启动。
- 恢复数据前。
- 工资月结后。
- 演示工作区重置前。

恢复流程会校验备份文件大小、备份类型、存储键和基本工作区结构，并要求用户再次确认覆盖当前数据。损坏恢复点当前不会出现在恢复点列表中，按编号读取时会失败。

## 操作建议

- 首次录入真实数据前，确认 Windows 账户隔离、磁盘访问权限和 BitLocker 状态。
- 设置 PIN 前先导出一份备份，并将其保存到受控的其他介质。
- 每次升级、迁移设备、批量调薪、批量调店或月结前后保留手动备份。
- 如需加密备份，使用“导出加密备份”并妥善保管口令；口令丢失无法自动恢复。
- 不要只依赖同一台电脑内的自动恢复点；设备损坏会同时影响工作区和本机恢复点。
- 不要通过公开 Issue、聊天记录、截图、日志或源码仓库传输备份、PIN、备份口令或真实工资数据。
- 月结后抽查正式导出与冻结快照；发现错误时记录原因后再解锁。

## 已知边界

本项目当前没有本地静态数据加密、云端冗余、多人权限、审计服务器、远程擦除、自动更新、signed Windows 发布通道或公开可信安装包。设备损坏、恶意软件、共享 Windows 账户、未加密磁盘和未加密备份均可能导致数据丢失或泄露。PIN 锁不能替代操作系统级账户隔离，手动加密备份的保护强度取决于口令强度。
