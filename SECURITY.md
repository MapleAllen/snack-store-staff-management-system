# 安全政策

## 支持范围

仅 `2.x` 最新版本接收安全修复。旧内部安装包不属于公开支持范围。

## 报告安全问题

请使用 GitHub 的 **Private vulnerability reporting** 私下提交，不要创建公开 Issue。报告应包含受影响版本、复现步骤、影响范围和缓解建议，但不得附带真实员工或工资数据。

## 数据与发布边界

- Windows 桌面版工作区保存在本机应用数据目录下的 `workspace.json` 文件中，文件本身未加密。应用依赖 Windows 账户隔离、磁盘加密（如 BitLocker）和物理访问控制。
- 桌面版支持 4-6 位数字 PIN 应用访问锁，PIN 通过 PBKDF2 哈希存储。连续 5 次错误输入触发 30 秒冷静期。
- 手动备份支持可选的口令加密（PBKDF2 + AES-256-GCM），导出后需正确口令方可恢复。
- 自动恢复点以明文 JSON 保存，不能替代异地备份。
- Web 开发预览（`npm run dev`）仅用于本地开发，不作为正式产品模式。
- 项目当前不发布 Windows 可执行文件。任何第三方提供的 EXE 都不是受信任的官方产物。
- 当前版本不提供"忘记 PIN"后的自动恢复机制；用户应在设置 PIN 前导出一份未加密备份。
- 备份口令和 PIN 不得通过公开 Issue、聊天记录或源码仓库传输。

## 已知安全边界

本项目没有本地静态数据加密、云端冗余、多人权限、审计服务器、远程擦除或自动更新。设备损坏、恶意软件、共享 Windows 账户和未加密备份均可能导致数据丢失或泄露。PIN 锁不能替代操作系统级账户隔离，加密备份的保护强度取决于口令强度。
