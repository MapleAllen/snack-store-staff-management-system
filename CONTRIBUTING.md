# 贡献指南

感谢你改进门店工资管理系统。提交更改前，请先确认改动没有引入真实员工、工资或门店经营数据。

## 开发流程

1. 使用 Node.js 22.12 或更高版本。
2. 运行 `npm ci` 安装锁定依赖。
3. 使用 `npm run dev` 开发 Web 界面，或使用 `npm run electron:dev` 验证桌面行为。
4. 提交前运行 `npm run check`。
5. 运行 `npm audit --audit-level=high`，不得引入高危或严重依赖漏洞。

## 变更要求

- 工资公式、数据迁移、月结与恢复逻辑必须附带测试。
- 保持 Web 和 Electron 两种运行环境可用；浏览器不支持的桌面能力必须有明确降级。
- 新增配置时同步更新迁移逻辑、备份校验和相关文档。
- 不要提交 `dist/`、`release/`、审计截图、日志、本机 `.npmrc` 或真实业务导出文件。
- 默认数据只能使用明确标识的虚构姓名和演示金额。
- 备份、导出和日志不得附加到公开 Issue；安全问题使用 Private vulnerability reporting。

## Pull Request

PR 应说明问题、实现方式、验证结果和仍未验证的平台边界。涉及界面变化时附一张当前截图；涉及 Windows 安装行为时说明是否已在真实 Windows x64 设备验证。
